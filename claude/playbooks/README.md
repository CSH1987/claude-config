# Playbooks — 작업유형 카탈로그 ("내가 일하는 법"의 증류본)

이 폴더는 **도메인에 무관한 공통 작업유형**을 어떻게 잘 처리하는지에 대한 재사용 가능한
플레이북 모음입니다. 설치하면 *빈 골격*이 아니라 이 "일하는 법"이 함께 옵니다.

## 왜 있나 (3가지 목적)
1. **노하우 전달 (기둥2)** — 개인 데이터(`claude-memory`, PRIVATE)는 전달되지 않지만, *방법*은
   여기(PUBLIC)에 살균·일반화되어 누구에게나 전달됩니다.
2. **콜드스타트 시드** — 새 사용자·새 분야는 축적이 0입니다. 이 카탈로그가 *출발 지식*이 됩니다.
3. **범용성 확장점 (기둥8)** — 분야(법률·의료·연구·사업…)는 달라도 *작업유형*(연구·기안·검토·결정)은
   공통입니다. 도메인 팩은 이 카탈로그 **위에** 얹습니다(현재는 확장점만; 선제 구축 안 함).

## 카탈로그 (현재)
| 플레이북 | 작업유형 | 한 줄 |
|---|---|---|
| [requirements.md](./requirements.md) | 요구사항 확정 | 만들기 전에 모호함을 1~3개 질문으로 닫는다 |
| [research.md](./research.md) | 리서치 | 다출처·교차검증·인용으로 신뢰 가능한 답을 만든다 |
| [decision.md](./decision.md) | 의사결정 | 결정을 근거와 함께 내리고 `decisions/`에 영구 기록한다 |
| [review.md](./review.md) | 검토 | "다 됐다" 선언 전에 적대적으로 검증한다 |

## 어떻게 쓰나
- Claude에게 "이 작업, 플레이북대로 해줘" 또는 "리서치 플레이북 적용" 처럼 말하면 됩니다.
- 또는 `claude/skills/playbooks/SKILL.md` 가 발견·적용 절차를 안내합니다.

## 어떻게 늘리나 (증식 — 이게 핵심)
플레이북은 **고정**이 아닙니다. 당신의 실제 노하우가 쌓이면 PUBLIC으로 승급해 키웁니다:
1. 작업 중 반복되는 *잘 통하는 방법*을 발견 → 성장루프(`/retro`)가 PRIVATE `_pending/`에 제안.
2. 충분히 일반적이고 PII가 없으면 → **`/promote`(hop2, 사람 검토)** 로 여기 PUBLIC 카탈로그에 추가.
3. **규칙: 개인정보·실명·고객/사건 정보는 절대 PUBLIC에 올리지 않습니다.** 일반화한 *방법*만.

> `claude/protocols/objective-function.md`, 결정 형식은 `claude/memory/SCHEMA.md` §2.
